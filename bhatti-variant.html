CORE.PRIMITIVES

Isolation without the overhead.

01

Real VMs. Not containers.

Every sandbox runs its own Linux kernel in a Firecracker microVM. Process isolation, filesystem isolation, network isolation. Not a namespace trick — a separate machine.

02

Memory snapshots. Not just filesystem.

When bhatti snapshots a VM, it captures everything: running processes, open file descriptors, TCP connections, in-memory state. Resume picks up exactly where it left off.

03

Three thermal states. Invisible to you.

Idle 30 seconds → warm (vCPUs paused, ~400µs resume). Idle 30 minutes → cold (snapshotted to disk, memory freed, ~50ms resume). Any API request transparently wakes it.

DEPLY.SELF_HOST

Run it on your hardware.

One command. Any Linux machine with KVM. Downloads pre-built binaries (~15MB), a kernel, and a minimal Ubuntu rootfs. No compilation, no Docker, no root during install. ~30 seconds.

Requirements

  • Linux (x86_64 or ARM64)
  • Hardware virtualization (/dev/kvm)
  • Root access for daemon (Firecracker req)

Tested on: Raspberry Pi 5, Hetzner AX-series, AWS Graviton metal.

# 1. Install binaries and default images
$ curl -fsSL https://bhatti.sh/install | sh

# 2. Start the daemon (requires root for kvm/network)
$ sudo bhatti serve
[INFO] Starting bhatti daemon v1.0.0
[INFO] Listening on /var/run/bhatti.sock

# 3. Create an API key in another terminal
$ sudo bhatti user create --name alice
bht_live_abc123def456...

# 4. Start creating VMs
$ export BHATTI_API_KEY=bht_live_...
$ bhatti create --name test

SYS.APPLICATIONS

Use cases

AI Coding Agents

Give LLMs a real, stateful environment to execute code, install packages, and run tests safely.

// Python SDK
import bhatti

sandbox = bhatti.create()
sandbox.exec("npm init -y")
sandbox.exec("npm install express")

res = sandbox.exec("node server.js &")
print(f"Server PID: {res.pid}")

sandbox.destroy()

Browser Automation

Boot a VM with Chromium pre-installed. Run Playwright scripts without polluting your host.

# CLI
$ bhatti create \
  --image ubuntu-playwright \
  --name scraper

$ bhatti exec scraper -- \
  npx playwright test

# Forward WS port to host
$ bhatti port-forward scraper 9222:9222

Docker Inside VMs

Run nested Docker daemon. Snapshot the entire VM with containers running to resume instantly later.

# CLI
$ bhatti create --image dind --name db
$ bhatti exec db -- docker-compose up -d

# Snapshot RAM and Disk
$ bhatti snapshot db --name pre-test

# Later: Resume in ~50ms
$ bhatti resume pre-test

Where bhatti is not the right choice:

  • You need managed global infrastructure (use Fly or E2B).
  • You need GPU passthrough for ML workloads (Firecracker doesn't support GPUs).
  • You need Windows VMs (Linux guests only).
  • You prioritize 10ms cold starts over absolute security isolation (use Docker).
